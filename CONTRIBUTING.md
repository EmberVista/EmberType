# Contributing to EmberType

Thank you for your interest in contributing to EmberType!

## How to Contribute

### Reporting Bugs

1. Check if the issue has already been reported in [GitHub Issues](https://github.com/EmberVista/EmberType/issues)
2. If not, create a new issue with:
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - macOS version and Mac model
   - EmberType version

### Suggesting Features

1. Open an issue with the "feature request" label
2. Describe the feature and its benefits
3. Explain your use case

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages (`git commit -m 'Add feature X'`)
6. Push to your fork (`git push origin feature/my-feature`)
7. Open a Pull Request

### Development Setup

1. macOS 14.0 or later
2. Xcode (latest version)
3. Apple Silicon Mac

See [BUILDING.md](BUILDING.md) for detailed build instructions.

## Code Style

- Follow Swift style guidelines
- Use meaningful variable and function names
- Keep functions focused and concise
- Add comments for complex logic

## Code of Conduct

Be respectful and constructive. We're all here to make EmberType better.

## Questions?

- Open an issue
- Email: contact@embertype.com

Thank you for contributing!
